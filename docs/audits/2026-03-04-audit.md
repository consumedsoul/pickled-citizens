## 🔴 Critical Issues (Fix Immediately)

### 1. Missing Content-Security-Policy (CSP) Header
**File:** `next.config.mjs`
**Lines:** 4-16
**Risk:** XSS attacks unmitigated — malicious scripts could execute in user browsers

The application now has X-Frame-Options, X-Content-Type-Options, Referrer-Policy, and X-DNS-Prefetch-Control, but is missing a Content-Security-Policy header. CSP is the primary browser defense against XSS. Without it, injected scripts (from compromised dependencies, stored XSS via user inputs, etc.) execute freely.

**Recommended Fix:**
```javascript
{ key: 'Content-Security-Policy',
  value: "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; font-src 'self' https://fonts.gstatic.com; connect-src 'self' https://*.supabase.co; img-src 'self' data: https:; frame-ancestors 'none'" }
```

Note: `'unsafe-inline'` for styles is needed for Tailwind's runtime styles. Fonts are loaded via `next/font` which inlines them, so `fonts.gstatic.com` may not be needed — verify after adding. The `script-src 'self'` may also need `'nonce-'` for Next.js hydration scripts in some configurations — test carefully.

**Why:** Five security headers now present, CSP is the most impactful missing one. This is the last piece of the security header suite.

### 2. No Audit Logging for Admin Profile PATCH Operations
**File:** `app/api/admin/users/route.ts`
**Lines:** 61-70
**Risk:** Admin profile edits leave no trace; impossible to audit who changed what and when

The PATCH handler updates user profiles with no audit event. The DELETE handler calls `admin_delete_user()` RPC which also doesn't log to `admin_events`. All other admin actions (role changes, session creation, user signup) are logged.

**Current code:** After line 68 (`return NextResponse.json({ ok: true })`), there is no audit logging.

**Recommended Fix:**
```typescript
// After the successful update:
await supabaseServiceRole.from('admin_events').insert({
  event_type: 'admin.user_profile_updated',
  user_email: callerEmail,
  payload: { target_user_id: body.userId, fields_updated: Object.keys(updatePayload) },
});
return NextResponse.json({ ok: true });
```

Similarly for DELETE — add before returning success:
```typescript
await supabaseServiceRole.from('admin_events').insert({
  event_type: 'admin.user_deleted',
  user_email: callerEmail,
  payload: { deleted_user_id: body.userId },
});
```

### 3. Hardcoded Admin Email in `middleware.ts`
**File:** `middleware.ts`
**Line:** 5
**Risk:** Two sources of truth for the admin email; changing it requires updating 2 files instead of 1

```typescript
// Current:
const ADMIN_EMAIL = 'hun@ghkim.com';

// Should be:
// Note: middleware.ts cannot use @/* path aliases (runs in Edge runtime before module resolution)
// Use process.env directly or import with relative path
import { ADMIN_EMAIL } from './src/lib/constants';
```

**Important caveat:** Next.js middleware runs in the Edge runtime with limited module support. The `@/` alias may not resolve. The safest fix is:
```typescript
const ADMIN_EMAIL = process.env.NEXT_PUBLIC_ADMIN_EMAIL ?? 'hun@ghkim.com';
```

And add `NEXT_PUBLIC_ADMIN_EMAIL=hun@ghkim.com` to `.env.local` and `wrangler.toml [vars]`. Alternatively, try a relative import first and see if the Edge runtime supports it.

**Why:** `constants.ts` is the documented single source of truth (`ADMIN_EMAIL`). The middleware currently duplicates it, creating a sync risk.

### 4. Missing `aria-current="page"` on Active Navigation Links
**File:** `src/components/Navigation.tsx`

>

### 5. DUPR Range Validation Missing in Admin User Edit UI
**File:** `app/admin/users/page.tsx`
**Lines:** ~169-185
**Risk:** Admin could set DUPR to invalid values (e.g., 99.9); UI accepts any float

The admin user edit form accepts `self_reported_dupr` as a freeform number input. DUPR ratings are bounded to 1.0–8.0. Setting out-of-bounds values creates data inconsistency.

**Recommended Fix:**
```typescript
if (dupr != null && (dupr < 1.0 || dupr > 8.5)) {
  setError('DUPR must be between 1.0 and 8.5');
  return;
}
```

Also applies to `app/profile/page.tsx` where the regex `/^\d{1,2}(\.\d{1,2})?$/` allows `99.99`.
