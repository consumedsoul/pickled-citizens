## 🟡 Medium Priority (Next Sprint)

### 1. `robots.txt` Advertises a Sitemap That Does Not Exist
**File:** `public/robots.txt:23`
**Risk:** Crawlers (including AI crawlers just enabled) request `/sitemap.xml` and get a 404, degrading discoverability signals we just added.

**Current:**
```
Sitemap: https://pickledcitizens.com/sitemap.xml
```

**Recommended Fix:** Create `app/sitemap.ts` using Next.js 14's built-in sitemap handler:
```typescript
import type { MetadataRoute } from 'next';

export default function sitemap(): MetadataRoute.Sitemap {
  const base = 'https://pickledcitizens.com';
  return [
    { url: `${base}/`, lastModified: new Date(), changeFrequency: 'weekly', priority: 1 },
    { url: `${base}/auth`, lastModified: new Date(), changeFrequency: 'monthly', priority: 0.6 },
    { url: `${base}/auth/signin`, lastModified: new Date(), changeFrequency: 'monthly', priority: 0.6 },
    { url: `${base}/leagues`, lastModified: new Date(), changeFrequency: 'weekly', priority: 0.7 },
    { url: `${base}/sessions`, lastModified: new Date(), changeFrequency: 'weekly', priority: 0.7 },
  ];
}
```

**Why:** Dynamic routes (`/leagues/[id]`, `/sessions/[id]`) are gated behind auth and should not be listed. Only public marketing pages belong in the sitemap.
